# 阶段 0 性能基线

- 测量日期：2026-07-13
- 硬件：MacBook Pro（Mac17,8），Apple M5 Pro，18 核，48 GB 内存
- 系统：macOS 26.5（Build 25F71），arm64
- 工具链：Node 24.16.0、npm 11.13.0、Rust/Cargo 1.96.1
- 前端：Vite 8.1.4 生产构建，通过本机 `vite preview` 提供
- 浏览器：Codex in-app Browser；本轮不把它等同于 Tauri 的 macOS WKWebView

## 生产前端交互基线

每个场景重新导航，使用 `?perf=1&bytes=N` 创建确定性文档，然后连续执行 100 个单字符输入。采样器按实际渲染帧合并输入，统计 `Transaction.time` 到下一帧的墙钟耗时。

| 文档   |   大小 | 帧样本 | 启动至编辑器挂载 |  p50 |  p95 |   p99 | 最大值 |
| ------ | -----: | -----: | ---------------: | ---: | ---: | ----: | -----: |
| 空文档 |    0 B |     56 |           2.3 ms | 2 ms | 4 ms |  4 ms |   4 ms |
| basic  | 32 KiB |     75 |          12.3 ms | 4 ms | 6 ms |  7 ms |   7 ms |
| medium |  1 MiB |     96 |          46.7 ms | 4 ms | 8 ms | 25 ms |  25 ms |

结论：`basic` p95 低于 8 ms，p99 低于 16.7 ms；`medium` p95 低于 16.7 ms。`medium` 出现一次 25 ms 的 p99/最大值尾部样本，阶段 1 仍需持续观察。

浏览器对 update listener 的计时分辨率为 1 ms，本次 100 个 callback 样本均记录为 0 ms；这只能说明监听器低于当前时钟分辨率，不能解释为零成本。

## 静置基线

空文档完成 100 字符输入后，连续静置 60 秒。测量报告字符串前后完全一致，样本数组没有新增，说明项目测量逻辑在无文档变化时没有重复工作。

随后启动 Tauri Release 原生窗口，使用 macOS `top` 每 5 秒采样一次，共测量两个连续 60 秒窗口：

| 窗口     | 12 个 `%CPU` 样本                              | 算术平均 |      内存 |
| -------- | ---------------------------------------------- | -------: | --------: |
| 暖机窗口 | 0、5.4、1.8、0、2.2、6.5、0.8、0.6、0、0、0、0 |    1.44% | 49–50 MiB |
| 稳定窗口 | 0、1.2、0、1.3、2.6、0、0、0、2.6、0.7、0、0   |    0.70% | 约 49 MiB |

稳定窗口的原始进程 CPU 平均值 0.70%，高于初始 0.5% 目标，因此不能宣称该目标通过。项目浏览器测量报告在独立 60 秒观察中完全不变，源码中也没有定时器或持续动画循环；当前数据尚不能判断剩余唤醒来自 Tauri/WKWebView 平台还是应用启动后的其他系统活动。需要用 Instruments Time Profiler 与 Energy Log 做归因，且补测后台窗口。原生 IME 候选窗也仍需人工回归。

## Vitest 合成事务基线

环境为 Node + jsdom，使用 Markdown language extension，在文档末尾执行 200 次单字符 `EditorState.update`。该结果只用于持续集成回归，不代表绘制延迟。

| --------------- | --------: | --------: | --------: |

## 构建观察

生产前端输出为 633.47 kB minified / 218.44 kB gzip，Vite 发出单 chunk 超过 500 kB 的警告。阶段 0 不为消除警告提前拆分功能；阶段 1 若启动或内存基线恶化，应优先检查 Markdown 默认语言支持带入的 HTML/代码语言模块。

## 空闲成本与释放

- 性能采样：仅文档变化时创建一个 RAF；空闲成本为无业务唤醒；销毁时取消 RAF；每类样本上限 500。
- Rust 日志：无后台线程或定时输出；Release 仅记录 warning/error。
- 图标生成：仅 Cargo build script 执行，运行时成本为零。
- 阶段 0 没有文件监听、Worker、Git、网络请求、轮询或持续动画循环。
