# ADR 001：编辑会话状态权威

- 状态：已接受
- 日期：2026-07-13
- 阶段：0

## 背景

编辑器需要在输入时保持低延迟，同时避免在 Svelte 和 Rust 中维护高频全文副本。多份可写文档状态会增加同步、版本冲突和内存复制成本。

## 决策

打开文档后，当前编辑会话的唯一权威文本状态是 `EditorState.doc`。Svelte 只创建和销毁编辑器容器，不接收每次文档变化，也不保存全文响应式副本。阶段 0 不在 Rust 中建立文档镜像。

未来文件打开会以初始快照创建 `EditorState`；保存会在明确的保存事件上读取完整快照。只有后续大文件基准证明保存快照复制是实际瓶颈后，才评估带 revision 的增量同步。

## 后果

- 输入、撤销、重做、选区和粘贴均由 CodeMirror 事务处理。
- Svelte 根组件不会因按键而响应式重渲染。
- Rust 不接收按键级 IPC。
- 保存和外部冲突处理必须以显式 revision/磁盘版本边界设计，不能引入隐式第二权威状态。

## 验证

`tests/editor/editor-session.test.ts` 验证编辑、选区、撤销、重做和 Unicode 文本。浏览器生产构建测试进一步验证 100 字符输入后撤销/重做的长度分别为 99/100。
